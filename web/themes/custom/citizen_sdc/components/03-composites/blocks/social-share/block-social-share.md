# Block Social Share SDC

The social share block is generated by the `citizen_custom` module, not by a
block content type or a view.

## Module pieces

- **Block plugin:** `web/modules/custom/citizen_custom/src/Plugin/Block/SocialShare.php`
  (plugin id `social_share`, admin label "Social Share")
- **Theme hook:** `social_share_block`, registered in
  `web/modules/custom/citizen_custom/citizen_custom.module` via
  `citizen_custom_theme()` with variables `title` and `data`

## Data flow

`SocialShare::build()` pulls the current node off the route match and returns
a render array of the form:

```php
[
  '#theme' => 'social_share_block',
  '#data'  => [
    'url'      => $node->toUrl()->toString(),
    'title'    => $node->getTitle(),
    'base_url' => $GLOBALS['base_url'],
  ],
]
```

When the block is being previewed or is placed on a route without a node
context, the plugin falls back to placeholder `example_url` / `Example Node
Title` values — so you'll see the block render in block-placement preview,
but the share URLs won't be real until it's on a node page.

## Template

`templates/blocks/social-share/social-share-block.html.twig` is the template
for the `social_share_block` theme hook. It does **not** extend
`block.html.twig` — Drupal's block rendering already wraps this theme hook's
output in `block.html.twig` automatically, so the outer `.block--social-share`
class comes in for free. The template just includes this SDC with the `data`
variable.

## Placement

Placement on news and event nodes is handled by layout builder. Any
node-context-specific placement styling (width, position on the page, etc.)
lives in the node-type SCSS files (`_news.scss`, `_event.scss`), not here.
This SDC's SCSS only covers the look of the block itself.
